Dokumentasi REST API

Smart Health API adalah REST API yang digunakan aplikasi Smart Health untuk mengelola data reservasi klinik, rekam medis, resep obat, rating dokter, dan statistik dashboard. Semua endpoint mengembalikan data dalam format JSON. Dibuat oleh: Eka Anastsya Situmorang — 241112564 (Tim ByteMe)

Base URL https://your-app.railway.app
🔒 Autentikasi: Semua endpoint (kecuali /auth/register dan /auth/login) memerlukan token JWT yang valid. Token dikirim otomatis melalui cookie httpOnly setelah login berhasil. Akses API langsung bisa menggunakan header Authorization: Bearer <token>. Setiap endpoint juga dijaga role-based authorization sesuai keterangan di masing-masing endpoint.

🔐 Autentikasi

POST /auth/register PUBLIC
FieldTipeKeterangan
usernamestringNama pengguna
emailstringEmail unik pengguna
passwordstringPassword min. 6 karakter, akan di-hash bcrypt
confirmPasswordstringHarus sama dengan password
// Response: redirect ke /pasien/dashboard dengan cookie JWT
POST /auth/login PUBLIC
FieldTipeKeterangan
emailstringEmail terdaftar
passwordstringPassword akun
// Response: redirect ke dashboard sesuai role (admin/dokter/pasien)
GET /auth/logout PUBLIC
// Menghapus cookie JWT dan redirect ke halaman login

⚗ Dokter

GET /api/doctors ALL ROLES
[{ "id": 1, "specialist": "Dokter Umum", "user": { "username": "dr. Budi", "email": "budi@..." }, "schedules": [...], "avgRating": "4.5", "jumlahRating": 3 }]
GET /api/doctors/:id ALL ROLES
{ "id": 1, "specialist": "Dokter Umum", "schedules": [...], "ratings": [...] }
POST /api/doctors ADMIN
FieldTipeKeterangan
usernamestringNama dokter
emailstringEmail login dokter
passwordstringPassword awal
specialiststringSpesialisasi dokter
PUT /api/doctors/:id ADMIN
FieldTipeKeterangan
specialiststringSpesialisasi baru (opsional)
usernamestringNama baru (opsional)
DELETE /api/doctors/:id ADMIN
{ "message": "Dokter berhasil dihapus." }
POST /api/doctors/:id/schedules ADMIN / DOKTER
FieldTipeKeterangan
dayOfWeekstringHari (Senin, Selasa, dst)
startTimestringJam mulai (format HH:MM)
endTimestringJam selesai (format HH:MM)
DELETE /api/doctors/schedules/:scheduleId ADMIN / DOKTER
{ "message": "Jadwal berhasil dihapus." }

📅 Appointment

GET /api/appointments ALL ROLES

Data otomatis difilter berdasarkan role: Pasien hanya melihat miliknya, Dokter melihat yang ditujukan kepadanya, Admin melihat semua.

GET /api/appointments/available ALL ROLES
Query ParamTipeKeterangan
doctorIdnumberID dokter yang dipilih
datestringTanggal (format YYYY-MM-DD)
{ "bookedSlots": ["2026-07-01T08:00:00.000Z", "2026-07-01T09:00:00.000Z"] }
POST /api/appointments PASIEN
FieldTipeKeterangan
doctorIdnumberID dokter yang dipilih
schedulestringTanggal & jam (ISO 8601: 2026-07-01T09:00:00)
keluhanstringKeluhan awal (opsional)
// 409 Conflict jika slot sudah terisi (validasi anti-bentrok aktif)
PUT /api/appointments/:id/status ALL ROLES
FieldTipeKeterangan
statusstringPENDING | DIKONFIRMASI | SELESAI | DIBATALKAN
// Pasien hanya bisa set DIBATALKAN. Dokter/Admin bisa set semua status.

📋 Rekam Medis

GET /api/health-records ALL ROLES

Otomatis difilter: Pasien hanya melihat miliknya, Dokter hanya yang dia buat, Admin melihat semua.

GET /api/health-records/:id ALL ROLES
{ "id": 1, "diagnosis": "Demam", "notes": "Pusing", "date": "...", "user": {...}, "doctor": {...}, "prescriptions": [...] }
POST /api/health-records DOKTER
FieldTipeKeterangan
userIdnumberID pasien
appointmentIdnumberID appointment terkait (opsional)
diagnosisstringDiagnosis dokter
notesstringCatatan medis (opsional)
{ "record": {...}, "recommendation": "Dokter Umum" } // recommendation: saran spesialis otomatis berdasarkan kata kunci diagnosis
